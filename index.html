# Editor Styles - IMGUI styling is handled by EditorStyles (assets) - Some nice functionalities: - Inheritance - Light and Dark Style share common properties - EditorStyles grouped by categories ----- # Editor Styles Tooling - Styles Editor ![editorstyles](img/editor-styles-editor.png) ----- # Editor Styles Tooling - Inspector ![editorstyles](img/editor-styles-inspector.png) ----- # IMGUI and StyleSheets - Align IMGUI styling with UIElements - Use StyleSheets as styling format - Some properties are common to IMGUI and UIElements (width, height, font) - Some properties are IMGUI specific: -unity-slice-left - Ensure all custom properties are prefixed with -unity ----- # Stylesheets Organization - Keep same folder structure as EditorStyles - Each folder has a dark.uss, light.uss and common.uss file - common.uss contains all properties common to both light and dark skin - light.uss/dark.uss mostly differs by tex-color and images (some offsets too!) ![stylesheets-demo](img/stylesheets.png) ----- # StyleSheets Improvements - We want to keep the same workflows as EditorStyles - Support for variable * { --text-color-normal: rgb(180, 180, 180); --text-color-light-shade: rgb(224, 224, 224); --text-color-lighter-shade: rgb(240, 240, 240); } .imgui-style-PlayerSettingsLevel { -text-color: --text-color-normal; } - Support for inheritance (extend) .imgui-style-PlayerSettingsLevel { -unity-extend: "Label"; /** overrides */ font: none; margin-bottom: 0; }
